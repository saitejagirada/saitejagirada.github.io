Before we start Transformer Architecture, let's build an intuition about how data is processed.

Think of a Transformer as a device that takes some input and spends some time learning from it during the training phase. After training, it becomes capable of understanding relationships between words and generating meaningful outputs.

Even if a Transformer is not trained, it can still generate output. However, the output will mostly consist of random and meaningless tokens. Training is what allows the model to learn useful patterns from data.

How Training Happens

If you have recently learned about Diffusion Models or CNNs, keep that knowledge aside for a moment and follow the flow. Transformers process information very differently from Convolutional Neural Networks.

Let's assume we have a text file as our dataset.

Computers cannot directly understand words. Therefore, the text is first converted into numerical values. These values are usually stored as vectors inside a data structure called a Tensor.

These tensors are then passed through the Transformer, where the model learns relationships between words, context, and meaning.
